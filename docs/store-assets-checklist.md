# Store Assets Checklist (Google Play + App Store)

Use este arquivo como checklist operacional para preparar screenshots e metadata.

**URLs publicas (Vercel):** [https://fitlocal.vercel.app/](https://fitlocal.vercel.app/) — landing `/landing`, privacidade `/privacy`, termos `/terms`.

## 1) Google Play

## Obrigatorios

- [ ] App icon: `512 x 512` PNG (32-bit, sem transparencias problemáticas)
- [ ] Feature graphic: `1024 x 500` PNG/JPG
- [ ] Screenshots de telefone (recomendado 6-8)
- [ ] Short description (ate 80 chars)
- [ ] Full description
- [ ] Categoria + tags
- [ ] Email de suporte
- [ ] Privacy Policy URL

## Sequencia sugerida de screenshots (alto impacto)

1. Tela principal (valor imediato)
2. Treino com timer
3. Progresso com grafico
4. Perfil e preferencias (idioma)
5. Privacidade / controle de dados
6. Fluxo de fotos de progresso

---

## 2) App Store (iOS)

## Obrigatorios (minimo pratico)

- [ ] App icon: `1024 x 1024` (sem alpha)
- [ ] Screenshots iPhone 6.7" (obrigatorio)
- [ ] Screenshots iPhone 6.1" (fortemente recomendado)
- [ ] Subtitle
- [ ] Description
- [ ] Keywords
- [ ] Support URL
- [ ] Privacy Policy URL
- [ ] App Review Notes

## Sequencia sugerida de screenshots (espelha Play)

1. Dashboard (resultado e clareza)
2. Workout (acao principal)
3. Progress (evidencia de valor)
4. Profile / Language / Privacy
5. Delete All Data (controle do usuario)

---

## 3) Regras de copy para evitar rejeicao

- [ ] Nao usar claims medicos (diagnosticar, tratar, curar)
- [ ] Enfatizar "educational estimates" / "fitness guidance"
- [ ] Manter promessa alinhada com o app real
- [ ] Citar privacidade local sem exagero tecnico

---

## 4) Checklist final antes de upload

- [ ] Assets batem com build atual
- [ ] Textos PT-BR e EN revisados
- [ ] URLs publicas funcionando:
  - [ ] `https://fitlocal.vercel.app/landing`
  - [ ] `https://fitlocal.vercel.app/privacy`
  - [ ] `https://fitlocal.vercel.app/terms`
- [ ] Nome do app final definido:
  - Recomendado: `FitLocal: Private Fitness Tracker`
